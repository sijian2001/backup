## 概要
`backup.sh` は指定されたディレクトリ内のファイルを年別または年月別に整理し、選択した方式で圧縮（または保持）するバックアップ用スクリプトです。処理中の進捗を標準出力へ表示し、既存のバックアップアーカイブがある場合は自動的にスキップします。

## 使い方
```bash
bash backup.sh <フォルダ名> <モード> <圧縮モード> [<開始年> <終了年>]
```

- `<フォルダ名>`: 処理対象ディレクトリ（絶対・相対パスどちらでも可）
- `<モード>`:
  - `1` 年単位（2010〜現在年）
  - `2` 年月単位（2010年1月〜現在年12月）
- `<圧縮モード>`:
  - `nozip` フォルダを残したまま整理
  - `zip` ZIP形式で圧縮（`zip` コマンドが必要）
  - `gzip` gzip圧縮（`back_<ID>.tar.gz` を生成）
- `[<開始年>]`: 開始年のみ指定すると、その年から現在年までを処理します（4桁必須）。
- `[<開始年> <終了年>]`: 開始年と終了年の両方を指定すると、その範囲に絞り込めます（省略時は 2010〜現在年）。

## 処理の流れ
1. 引数を検証し、対象ディレクトリに移動します。
2. 指定モードに応じて開始年〜終了年の年または年月を走査します。
3. 該当するファイル（ファイル名に対象年／年月を含む）を専用フォルダへ移動します。
4. フォルダ内にファイルが存在する場合のみ圧縮モードに従って処理します。
   - `zip` : アーカイブ作成後に元フォルダを削除します。
   - `gzip`: `tar.gz` を作成し、成功時に元フォルダを削除します。
   - `nozip`: フォルダを残します。
5. 対象ファイルが存在しない場合はフォルダを削除して次へ進みます。

## テスト
簡易検証用に `test_backup/run_tests.sh` を用意しています。  
`bash test_backup/run_tests.sh` を実行すると `nozip`, `zip`, `gzip` の各モードを通しで確認できます。
